# EcoVault 贡献指南

感谢你参与 EcoVault（生态保险箱）建设。提交贡献前，请先阅读 [docs/开发规范.md](docs/开发规范.md)。

## 开发准备

- 安装 Java 25。
- 安装 Maven 3.9+。
- 确认本地使用 UTF-8 编码。

## 构建与测试

```bash
mvn clean package
mvn test
```

覆盖率报告位置：

```text
target/site/jacoco/index.html
```

## 分支规范

分支名必须以用户名开头：

```text
<用户名>/<类型>/<简短主题>
```

示例：

```text
unknowIfGuestInDream/feat/password-tags
```

## 提交规范

提交信息建议使用中文并遵循格式：

```text
<类型>: <中文简短说明>
```

示例：

```text
fix: 修复单设备登录会话失效问题
```

## Pull Request 要求

- 已更新相关 `docs/` 文档。
- 已添加或更新必要单元测试。
- 已通过本地构建与测试。
- 未包含密钥、Token、真实密码、数据库文件或生产配置。
- 涉及安全、数据库、部署或 API 的变更已说明影响范围。

## 文档与测试

每次功能、接口、数据库、安全策略或部署流程变更，都必须同步更新文档。新增业务逻辑应补充 JUnit 5 测试，核心安全逻辑应覆盖正常、异常与边界场景。

## 安全提醒

禁止提交真实敏感信息。示例配置必须使用占位符。发现安全漏洞时，请通过 GitHub 安全公告私密反馈渠道报告。
